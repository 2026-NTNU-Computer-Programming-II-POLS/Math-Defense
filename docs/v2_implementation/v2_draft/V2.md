## 1. Overview & Scope

Math Defense is an educational tower-defense game where paths, towers, and
combat mechanics are all grounded in mathematical concepts. V2 is a major
redesign that replaces the current V1 systems with a richer, more
math-integrated experience and adds a multi-user classroom layer.

### V2 Pillars

| Pillar | Description |
|--------|-------------|
| **Math-as-gameplay** | Every tower type maps to a specific math domain (equations, radar/trig, matrices, limits, calculus). Paths are real mathematical curves. |
| **Risk/reward choices** | Limit towers can heal enemies; Calculus results can destroy your own tower; Initial Answer determines intel visibility. |
| **Classroom integration** | Teacher/Student/Admin roles, class management, Grabbing Territory mode for intra-class and inter-class competition. |
| **Star-based difficulty** | 1-5 star system derived from path function composition, driving difficulty curves and territory value. |

## 3. Game Modes

### 3.1 Individual Mode

Standard single-player tower defense. Player progresses through levels of
increasing difficulty. Levels are organized by star rating. Each level has a
unique combination of path functions (see Section 4), waves, and enemies.

Flow:

```
Select Level (star-rated)
  → Initial Answer phase
  → Wave 1..N (with Monty Hall events between waves)
  → Level End → Score calculation → Global Ranking update
```

Individual mode also feeds into:
- **Achievement** unlocks
- **Talent** point earning
- **Global Ranking**

## 4. Path System

### 4.1 Overview

V2 replaces V1's piecewise segment-based paths with **full mathematical curve
paths** that span the entire map. Each path is a single continuous function.
Enemies follow these curves from their spawn point (map border intersection)
toward the shared endpoint (the intersection of all paths).

### 4.2 Path Types

| ID | Path Type | Function Family | Examples |
|----|-----------|----------------|----------|
| **A** | Equation | Polynomials of degree 1, 2, or 3 | `y = 2x + 1`, `y = x^2 - 3x + 2`, `y = x^3 - x` |
| **B** | Trigonometric | sin, cos functions | `y = 2sin(x) + 1`, `y = cos(2x)` |
| **C** | Logarithmic | log functions | `y = ln(x) + 3`, `y = 2log(x-1)` |

Sub-types within A-path:

| Degree | Name |
|--------|------|
| 1 | Linear |
| 2 | Quadratic |
| 3 | Cubic |

### 4.3 Path Groups (Combinations)

Each level uses a **group** of k paths drawn from the path types:

| Group ID | Composition | Path count (k) |
|----------|-------------|-----------------|
| type-1 | {A-path only} | n = 2 to 5 |
| type-2 | {A-path, B-path} | n = 2 to 3 |
| type-3 | {A-path, C-path} | n = 2 to 3 |
| type-4 | {B-path, C-path} | n = 2 to 3 |
| type-5 | {B-path only} | n = 2 to 3 |
| type-6 | {C-path only} | n = 2 to 3 |
| type-7 | {A-path, B-path, C-path} | n = 3 |

### 4.4 Start Points (Spawn Points)

Each path intersects the map boundary at one or more points. These boundary
intersections become **enemy spawn points**. Different paths can share a spawn
point or have distinct ones.

The number of spawn points is a key difficulty driver — more spawn points mean
enemies approach from more directions.

### 4.5 End Point (Shared Destination)

All paths in a level converge at a single shared intersection point — the
**endpoint** that enemies march toward (equivalent to the "base" in classic
tower defense).

**Generation strategy** (reverse construction):
1. Pick the endpoint P(x0, y0) first.
2. Generate all path functions constrained to pass through P.
3. Numerically verify there are no other shared intersections of ALL paths.
4. Choose an interval [a, b] around x0 containing exactly one intersection.
5. Present [a, b] to the player during the Initial Answer phase.

**Constraint**: At the endpoint, the slopes of different path functions should
differ enough to avoid "near-tangent" situations that would make the interval
impractically small.

### 4.6 Legal Tower Placement

All towers in V2 are placed on **grid intersection points** (lattice points),
not inside grid cells. Legal placement positions are **randomly generated**
on the map, constrained to **not overlap any path**. A minimum clearance
distance from paths should be enforced.

## 5. Difficulty Rating (Star System)

Levels are rated 1-5 stars based on the composition of their path functions.
The notation `{a, b, c, ...}` describes a multiset of polynomial degrees
and/or function types in the level.

### 5.1 Star Definitions

#### 1 Star — Beginner Training (Pure Linear)
All paths are degree-1 (straight lines). Very few spawn points, simple logic.

- k=2: {1,1}, {1,2}, {2,2}
- k=3: {1,1,1}
- k=4: {1,1,1,1}

#### 2 Stars — Intermediate (Simple Logic Switching)
Player must switch between linear and quadratic thinking, or handle one
transcendental function alongside polynomials.

- k=2: {1,3}, {2,3}, {3,3}
- k=3: {1,1,2}, {1,2,2}, {2,2,2}
- k=4: {1,1,1,2}, {1,1,2,2}
- k=5: {1,1,1,1,2}, {1,1,1,1,1}
- Single transcendental: {1,log}, {2,log}, {3,log}, {1,sin}, {2,sin}, {3,sin},
  {1,cos}, {2,cos}, {3,cos}

#### 3 Stars — Advanced (Trinity Challenge)
Multiple polynomial degrees appear, with 3-4 spawn points. Or one
transcendental function paired with two polynomials.

- k=3: {1,1,3}, {1,2,3}, {2,2,3}
- k=4: {1,1,1,3}, {1,2,2,2}, {2,2,2,2}
- k=5: {1,1,1,2,2}, {1,1,2,2,2}
- Dual-component: {1,1,log}, {1,2,log}, {1,3,log}, {2,2,log}, {2,3,log},
  {3,3,log}, {1,1,sin}, {1,2,sin}, {1,3,sin}, {2,2,sin}, {2,3,sin},
  {3,3,sin}, {1,1,cos}, {1,2,cos}, {1,3,cos}, {2,2,cos}, {2,3,cos},
  {3,3,cos}

#### 4 Stars — Expert (High-Frequency Mix)
4-5 spawn points, intense calculation pressure. Two transcendental functions
or high-degree plus transcendental.

- k=3: {1,3,3}, {2,3,3}, {3,3,3}
- k=4: {1,1,2,3}, {1,1,3,3}, {1,2,2,3}, {1,2,3,3}, {2,2,2,3}, {2,2,3,3}
- k=5: {1,1,1,2,3}, {1,1,1,3,3}, {1,2,2,2,2}, {2,2,2,2,2}
- Dual transcendental: {log,log}, {sin,sin}, {cos,cos}, {log,sin},
  {log,cos}, {sin,cos}
- Triple-component: {1,log,log}, {2,log,log}, {3,log,log}, {1,sin,sin},
  {2,sin,sin}, {3,sin,sin}, {1,cos,cos}, {2,cos,cos}, {3,cos,cos},
  {1,log,sin}, {2,log,sin}, {3,log,sin}, {1,log,cos}, {2,log,cos},
  {3,log,cos}, {1,sin,cos}, {2,sin,cos}, {3,sin,cos}

#### 5 Stars — Legendary (Ultimate Hell)
5+ spawn points, no linear paths as buffer. Triple transcendental
combinations cause extreme visual and cognitive overload.

- k=4: {1,3,3,3}, {2,3,3,3}, {3,3,3,3}
- k=5: {1,1,3,3,3}, {1,2,3,3,3}, {2,2,3,3,3}, {1,1,2,3,3}, {1,2,2,3,3},
  {2,2,2,3,3}, {1,3,3,3,3}, {2,3,3,3,3}, {3,3,3,3,3}, {1,1,2,2,3},
  {1,2,2,2,3}, {2,2,2,2,3}
- Triple transcendental: {log,log,log}, {sin,sin,sin}, {cos,cos,cos},
  {log,log,sin}, {log,log,cos}, {sin,sin,log}, {sin,sin,cos},
  {cos,cos,log}, {cos,cos,sin}, {log,sin,cos}

### 5.2 Star Value

The star rating directly determines the **territory value** in Grabbing
Territory mode (see Section 13). A 5-star territory is worth 5 points; a
1-star territory is worth 1 point.

## 6. Initial Answer Mechanism

Before each level begins, the system displays all path functions and an
interval [a, b], then asks the player to find the unique legal intersection
point of all paths within that interval (i.e., the endpoint / base location).

### 6.1 Outcomes

| Player Action | IA Value | Path Display During Level | Score Impact |
|--------------|----------|--------------------------|-------------|
| **Answer correctly** | 1 | All paths and endpoint shown throughout | Best score multiplier |
| **Answer incorrectly** | 0 | All paths and endpoint shown throughout | Score penalized |
| **Pay money to skip** | 0 | All paths and endpoint shown throughout | Score penalized + gold spent |
| **Ignore (no answer until level ends)** | 0 | **Paths and endpoint completely hidden** | Score penalized + blind play |

### 6.2 Design Intent

- The first three options all grant full path visibility. The only difference
  is the score multiplier (IA=1 vs IA=0) and gold cost.
- **Ignoring** is the heaviest punishment: the player must play the entire
  level without seeing any paths or the endpoint, essentially "blind" tower
  defense. This strongly incentivizes engaging with the math.

### 6.3 Answer Format

The answer is presented as a **multiple-choice question** with several
candidate intersection points. The player selects one. Incorrect selection
results in IA=0 but still grants path visibility.

## 7. Defense Towers

V2 has **5 tower types**, all upgradeable via the Talent system.

### 7.1 Magic Tower

| Property | Value |
|----------|-------|
| **Math domain** | Equations (degree 1/2/3), Trigonometric, Logarithmic |
| **Behavior** | The assigned function is **drawn directly onto the map** as a visible curve. The curve acts as a **zone of effect** — the player chooses whether it serves as an **enemy debuff zone** or an **allied tower buff zone**. |
| **Function assignment** | Each Magic tower is bound to one function (n = 1). The function determines the shape and position of the zone on the map. |
| **Mode** | Player selects one of two modes per tower: (1) debuff enemies passing through the curve, or (2) buff allied towers located on or near the curve. |
| **Upgradeable** | Yes (via Talent) |

### 7.2 Radar Tower

| Property | Value |
|----------|-------|
| **Math domain** | Trigonometry / radian intervals |
| **Sub-types** | Type-A, Type-B, Type-C (purchased separately, **cannot switch** between types, different prices) |
| **Base range** | All three types have a **circular attack range** by default. |
| **Radian interval** | Player can configure a **radian interval** (arc sector) within the circle (with an upper-bound cap). Within this arc, the tower gains **attack speed and/or damage bonuses**. Optionally, the tower can be restricted to **only attack within that arc**. |

| Sub-type | Analogy | Targeting | Attack Speed | Damage | Behavior |
|----------|---------|-----------|-------------|--------|----------|
| **Type-A** | Flamethrower / wiper | No target selection | — | AoE | Sweeps continuously around the full circle; damages all enemies it passes over |
| **Type-B** | Basic tower | Can select target | High | Low | Classic fast-firing single-target tower |
| **Type-C** | Sniper tower | Can select target | Low | High | Slow, powerful single-target shots |

**Upgradeable attributes** (via Talent):
- Type-A: area-of-effect (range)
- Type-B/C: attack speed, damage, target quantity per attack

### 7.3 Matrix Tower

| Property | Value |
|----------|-------|
| **Math domain** | Linear algebra (dot product / matrix multiplication) |
| **Pairing** | Two Matrix towers form one effective group |
| **Range** | Each tower has a predefined range MR |
| **Overlap zone** | The intersection area of both towers' ranges is the attack zone |
| **Targeting** | Can select targets within the overlap zone |
| **Attack style** | Laser lock on a single target; damage increases over time |

**Damage calculation**:

Each tower's grid-point coordinates become a vector. Tower A at (x₁, y₁)
produces row vector `[x₁, y₁]` (1×2). Tower B at (x₂, y₂) produces column
vector `[x₂, y₂]ᵀ` (2×1). The **base damage** is the scalar product:

```
base_damage = [x₁, y₁] × [x₂, y₂]ᵀ = x₁·x₂ + y₁·y₂
```

Example: Tower A at (1, 2), Tower B at (2, 1) → 1×2 + 2×1 = **4 base damage**.

The laser locks onto one target; damage **increases over time** from this
base value as long as the laser stays connected.

**Upgradeable attributes** (via Talent): range MR, target quantity per attack

### 7.4 Limit Tower

| Property | Value |
|----------|-------|
| **Math domain** | Limits |
| **Placement** | Tower is placed at x = a on the map |
| **Mechanic** | Player is presented a multiple-choice question: compute lim f(x)/(x-a) as x→a. Based on the selected answer: |

| Result | Effect | Damage Ordering |
|--------|--------|----------------|
| **+infinity** | Deals damage to enemies (maximum) | D(+inf) > D(+C) |
| **Positive finite (+C)** | Deals damage to enemies (moderate) | |
| **0** | Tower is **removed** — no refund | |
| **Constant (non-variable)** | Tower has **no effect** (disabled) | |
| **Negative finite (-C)** | **Heals enemies** (moderate) | H(-inf) > H(-C) |
| **-infinity** | **Heals enemies** (maximum) | |

**Risk/reward**: Choosing wrongly can heal enemies or destroy your tower.
The question format is multiple-choice — the player picks from options.
(Original design also allows fill-in-the-blank as a harder variant.)

**Upgradeable**: Yes (via Talent)

### 7.5 Calculus Tower

| Property | Value |
|----------|-------|
| **Math domain** | Differentiation and integration |
| **Setup** | System presents 3 to 5 **preset functions** (player cannot input custom functions). Player selects one function, then chooses an operation: first derivative, second derivative, or integral. |
| **Output** | The result `C * x^n` determines pet spawning (see below). Pets are companion units that fight autonomously within the tower's effective range. |
| **Function change** | Each wave, the player can pay money to apply another derivative or integral to the **current result**, chaining transformations across waves. |

#### Result interpretation: `C * x^n`

The **coefficient C** determines pet quantity; the **exponent n** determines
pet behavior.

| Exponent (n) | Pet Trait |
|-------------|-----------|
| **1** | **Slow-down** (slows enemies) |
| **2** | **High attack speed** |
| **3** | **High damage** |
| **n > 3 or n < 0** | No special trait, but pets can still **basic-attack** |
| **0 (constant)** | Tower has **no effect** (disabled, no pets) |
| **result = 0** | Tower is **removed** (no refund) |

#### Coefficient (C) rules

| Coefficient | Pets spawned | Ability modifier |
|-------------|-------------|-----------------|
| **Integer C** (e.g., 4) | C pets | Full strength (1.0x) |
| **Fraction C** (e.g., 1/4) | 1 pet | Overall ability multiplied by C (weakened) |

#### Example: chaining across waves

| Wave | Action | Current function | Result | Pets |
|------|--------|-----------------|--------|------|
| 1 | Select `2x^2`, apply 1st derivative | — | `4x^1` | 4 slow-down pets |
| 2 | Pay money, apply derivative again | `4x^1` | `4` (constant) | **Disabled** — no pets |

Because the system only offers preset single-term functions, the result is
always a monomial `C * x^n` — multi-term polynomials cannot occur.

**Upgradeable attributes** (via Talent): pet attack speed, pet damage

### 7.6 In-Game Tower Upgrades

Separate from the persistent Talent system (Section 12.2), each placed tower
can be upgraded **during a level** by spending gold. These upgrades are
**temporary** — they reset when the level ends.

| Level | Upgrade Cost (% of base price) | Generic Stat Bonus |
|-------|-------------------------------|-------------------|
| 1 (base) | — | — |
| 2 | 60% | +25% damage, +10% range |
| 3 | 100% | +50% damage, +20% range, +15% attack speed |

Tower-type-specific bonuses stack on top of the generic table:

| Tower | Level 2 Bonus | Level 3 Bonus |
|-------|--------------|--------------|
| **Magic** | +zone width | +effect strength |
| **Radar A** | +sweep speed | +AoE width |
| **Radar B** | +target count (1→2) | +target count (2→3) |
| **Radar C** | +crit chance | +crit damage |
| **Matrix** | +damage ramp rate | +laser can hit 2 targets |
| **Limit** | +damage multiplier | +range |
| **Calculus** | +pet HP | +pet count (+1) |

In-game upgrades and Talent upgrades stack **multiplicatively**:

```
effective_stat = base_stat
  * (1 + in_game_upgrade_bonus)
  * (1 + talent_bonus)
```

## 8. Enemies

### 8.1 Standard Enemy Types

| Type | Description |
|------|-------------|
| **General** | Baseline enemy. Normal HP, speed, reward. |
| **Split** | Splits into smaller enemies on death. |
| **Strong** | High HP, slow speed, high reward. |
| **Fast** | Low HP, high speed. |
| **Helper** | Support unit — assists other enemies (buff/heal nearby enemies). |

### 8.2 Boss Types

#### Boss Type-A
- **Shield**: Has a protective shield that must be broken first.
- **High health**: Significantly more HP than standard enemies.
- **Minion generation**: Periodically spawns smaller enemies (minions).

#### Boss Type-B
- **Includes all Type-A abilities** (shield, high health, minion generation).
- **Chain Rule challenge**: f(g(x))' mechanic.

Boss Type-B flow:

```
Boss Type-B appears
  → System displays: f(g(x))' = ?
  → Multiple-choice answer presented to player
  │
  ├── Correct answer
  │   → Boss immediately splits into two enemies:
  │     f'(g(x)) and g'(x)
  │
  └── Incorrect answer
      → Must beat the original boss first (full HP)
      → THEN boss splits into: f'(g(x)) and g'(x)
```

Answering correctly is a significant advantage: the split enemies have less
total HP than the original boss. Answering incorrectly means fighting the
full boss PLUS dealing with the split afterward.

## 9. Wave Events (Monty Hall)

### Trigger Condition

The Monty Hall event does **not** occur after every wave. It triggers
**progressively** when the player's cumulative **kill value** (sum of all
killed enemies' individual values) reaches certain thresholds. Different
enemies have different kill values, so the trigger pacing depends on the
enemy composition the player faces and defeats.

### Flow

1. **3 to 5 doors** are displayed. Exactly one contains a **buff**; the rest
   are empty.
2. Player **chooses one door** (without opening it).
3. System **reveals and removes one empty door** from the unchosen doors.
4. System asks: **"Do you want to switch?"**
5. Player decides to keep or switch.
6. Chosen door opens — player receives the buff (or nothing).

### Design Intent

This is the classic Monty Hall problem. Statistically, switching gives a
higher probability of winning the buff. Over many waves, players who
understand the probability gain a meaningful advantage. This reinforces the
educational math theme.

## 11. Scoring Mechanism

### 11.1 Formulas

```
S1 = kill / [time_total - n * sum(time_exclude_prepare)]
S2 = kill / cost_total
K  = MAX(m * S1 + (1 - m) * S2),  where 0.5 <= m <= 0.7
```

- **S1** measures **time efficiency**: kills per unit of active combat time
  (total time minus preparation phases).
- **S2** measures **cost efficiency**: kills per unit of gold spent.
- **K** is the weighted combination, optimized over m in [0.5, 0.7].

### 11.2 Total Score

```
Total Score = K ^ (1 / [1 + (2 + health_origin - health_final - Initial Answer)])
```

Where:
- `Initial Answer` = 1 if the player answered correctly, 0 otherwise
  (False / Money / Ignore all yield 0).
- The exponent `1 / [1 + (2 + hp_origin - hp_final - IA)]` rewards
  players who take less damage and answer the Initial Answer correctly.

**Example values** (assuming `health_origin = 20`):

| Scenario | hp_final | IA | Exponent | Effect |
|----------|----------|----|----------|--------|
| Perfect (no damage, correct) | 20 | 1 | 1/(1+2+0-1) = 1/2 | K^0.5 (square root — best) |
| No damage, wrong/money/ignore | 20 | 0 | 1/(1+2+0-0) = 1/3 | K^0.33 |
| Lost 10 HP, correct | 10 | 1 | 1/(1+2+10-1) = 1/12 | K^0.083 |
| Lost all HP, wrong | 0 | 0 | 1/(1+2+20-0) = 1/23 | K^0.043 (worst) |

A higher exponent yields a higher Total Score (for K > 1), so the formula
naturally rewards both survivability and math engagement.

### 11.3 Variables

| Variable | Meaning |
|----------|---------|
| `kill` | **Cumulative kill value** — sum of individual kill values of all enemies defeated. Different enemy types have different values (e.g., boss > strong > general). This is NOT a simple kill count. |
| `time_total` | Total elapsed time from level start to end |
| `time_exclude_prepare` | Time spent in preparation phases (not combat) |
| `n` | Number of preparation phases |
| `cost_total` | Total gold spent during the level |
| `m` | Weight parameter, optimized in [0.5, 0.7] |
| `Initial Answer` | 1 if answered correctly, 0 otherwise |
| `health_origin` | Starting HP |
| `health_final` | Ending HP |

## 12. Achievement & Talent System

### 12.1 Achievements

Achievements are unlocked by meeting specific conditions during gameplay
(e.g., complete a 5-star level, kill N enemies, survive N waves without
losing HP). Each unlocked achievement grants **talent points**.

### 12.2 Talent Tree

Talent points are spent to permanently upgrade tower attributes. The talent
tree connects to all five tower types:

| Tower | Upgradeable Attributes |
|-------|----------------------|
| **Magic** | Damage, range, attack speed (specific attributes TBD) |
| **Radar Type-A** | Area-of-effect (range) |
| **Radar Type-B/C** | Attack speed, damage, target quantity per attack |
| **Matrix** | Range MR, target quantity per attack |
| **Limit** | Damage multiplier, range (specific attributes TBD) |
| **Calculus** | Attack speed, damage |

Talent upgrades are **persistent** — they carry across all game sessions.

## 13. Grabbing Territory

### 13.1 Overview

Grabbing Territory is a competitive mode where students compete to seize and
hold **limited territory** (levels/stages). It operates both **within a class**
(intra-class) and **between classes** (inter-class).

### 13.2 Configuration (Teacher)

| Setting | Description |
|---------|-------------|
| **Deadline** | Date/time when the activity ends and territories are finalized |
| **Total levels** | Number of territory slots available (finite and limited) |
| **Levels content** | The specific level configurations (difficulty, path combinations) for each territory |

### 13.3 Mechanism

```
Student selects a territory (level) and plays it
  │
  ├── Lose (fail to complete the level)
  │   └── Fail — no territory change
  │
  └── Win (complete the level, Total Score recorded)
      │
      ├── Territory is UNOCCUPIED
      │   └── Seize territory — student occupies it with their score
      │
      └── Territory is OCCUPIED (by another student)
          │
          ├── score_new > score_old
          │   └── Seize territory — new student takes over
          │
          └── score_new <= score_old
              └── Fail — occupying student retains territory
```

### 13.4 Rules

| Rule | Detail |
|------|--------|
| **Max territories per student** | 5 (hard cap) |
| **Counter-seize** | Unlimited. A student whose territory was taken can re-play and reclaim it if they beat the new holder's score. |
| **Strategic pressure** | Because of the 5-territory cap, skilled students are incentivized to hold fewer high-star territories rather than many low-star ones. |
| **Territory value** | Equal to the level's star rating (1-5 points per territory). |
| **Scope** | A single Grabbing Territory activity can be intra-class (students within same class) or inter-class (students across classes). |

### 13.5 Settlement

When the deadline arrives:
- All territories are frozen.
- Each student's total territory value = sum of star ratings of occupied
  territories.
- Rankings are computed (see Section 14).

## 14. Ranking System

### 14.1 Class Ranking

- **Scope**: Within a single class.
- **Metric**: Player's Total Score (from Individual mode or Grabbing
  Territory).

### 14.2 Global Ranking

- **Scope**: All players across all classes.
- **Metric**: Player's Total Score.

### 14.3 Internal & External (for Grabbing Territory)

| Type | Scope | Metric |
|------|-------|--------|
| **Internal** | Cross-class, per-student | The individual player's Total Score |
| **External** | Cross-class, per-class | The **average** Total Score of all students in the class |

This dual ranking allows both individual standouts and well-performing classes
to be recognized.
